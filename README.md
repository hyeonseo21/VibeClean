(11/08)
--> 이 날은 기본적으로 postman으로 데이터 저장하고 get하던 방식을 벗어나서 mosquitto이용해서 값 받고 그리고 받은 값을 바로 해당하는 table에 저장하도록 바꾸어주었다. 
--> 이렇게 하기 위해서 yml과 build.gradle에 의존성을 추가하고 환경 변수 설정을 해주었으며, 별도의 MQTT 설정 클래스와 서비스 계층 파일 등을 두었다. 
--> 참고로 기본적인 기능을 하던 api들을 이제 datetime을 table에 추가하거나 센서 값을 원래 int로 설정했었는데 double로 바꾸거나 이런 변화를 주었다. 
--> master 브랜치에 저장된 것이 해당 내용이다. 

(11/09)
--> 3초동안 신호를 받지 않으면 sensor와 status 테이블을 초기화하도록 바꾸어주었다. (이 기능을 하는 메서드는 1초마다 실행되게 만들었다. )
--> total 시간 계산하는 것도 만들었다. 
--> 또한 GET API들에게 만약에 비어있는 경우라면, 기본 값을 주도록 그렇게 수정해야 한다. 
--> 또 POST API에게 값을 받게 되면 그떄는 이제 STM쪽으로 publish바로 하도록 그것 역시 서비스 계층에 만들고자 한다.
--> 그 외에도 명세서 간단하게 타입 맞추고, 변수명 통일 시키고, 또 reqeust response에 맞게 하도록 postman이용해서 확인했다.
--> 그 외에 자잘한 오류나 놓쳤던 부분도 수정하였다. 
--> 또 기본적인 깃헙과 깃의 관리법에 대해 정리하였으며, 이를 활용해 실제로 바꾼 코드를 깃으로 add, commit, push하였으며 브랜치 관리 역시 시작하였다. 
--> main브랜치에는 나중에 최종적으로 백엔드 코드 다 마무리 한 후에 해당 결과를 넣을 예정이며, master 브랜치에는 현재까지 진행된 결과를, 그리고 해당 값을 기준으로 work 브랜치에서 작업하며, 오늘의 개발 분량이 다 끝나면 최종 확인 후 master에 올릴 예정이다. 다음 날은 계속 master에서 시작해서 마저 개발을 진행할 예정이다. 
--> 오늘 여기까지 하고 나면 백엔드 쪽은 더 건들 것이 없다. 11월 10일에는 STM쪽에서 wifi 모듈 설정에 대해서 처리하고 11월 11일에는 STM쪽에서 publish, subscribe하는 코드를 작성할 예정이다.
